## Purpose

Universal Dragon Core is the main public architecture and documentation base for the Universal Dragon ecosystem.

It keeps the project direction clear, organized, and safe while the real private systems continue on the Pi5/server side.

This repository should explain the full roadmap without exposing private backend details, API keys, tunnel credentials, private IP addresses, or live internal logs.

## Main Project Roles

### Project One — NOVA Guardian Core

NOVA Guardian Core is the safe brain-first base.

It focuses on:

- approval-first actions
- backup before changes
- validation before applying patches
- rollback on failure
- event logs, error logs, and learn logs
- defensive diagnostics and repair planning

### Project Two — Universal Dragon Studio

Universal Dragon Studio is the creator-tools layer.

It focuses on:

- website builder planning
- app builder planning
- video/photo workflow planning
- long-video to shorts direction
- thumbnail, captions, titles, and descriptions
- safe build queue with approval-first workflow

### Project Three — Dragon Eye

Dragon Eye is the future vision and camera layer.

It focuses on:

- camera feeds
- OpenCV experiments
- image understanding
- event detection
- future robotics perception support

### UDOS — Assistant-Based Control Layer

UDOS is the long-term system concept.

It is not a normal app-based operating system. It is an assistant-first control layer where EVE/NOVA coordinates tools, memory, files, devices, services, and approvals.

Core idea:

```text
Phone is not app-based.
Phone is assistant-based.
```

## EVE Role

EVE is the front-facing companion/interface layer for Universal Dragon.

EVE can become:

- web UI
- terminal interface
- assistant layer
- command bridge
- memory interface
- mobile/launcher prototype later

EVE should always stay safe, approval-based, and reversible.

## Safety Boundary

Public repos are for:

- architecture
- docs
- public websites
- safe frontend code
- project history
- non-sensitive planning

Private systems must stay private:

- `.env` files
- API keys
- Groq/OpenAI keys
- Cloudflare tunnel credentials
- SSH keys
- Pi private IPs
- live backend logs
- uploads and render outputs
- unrestricted terminal controls

## Engineering Rules

Every Universal Dragon project should follow these rules:

1. Do not restart from zero without reason.
2. Backup before editing important files.
3. Use rollback capability where possible.
4. Include an error handler in real apps.
5. Keep public files clean and safe.
6. Prefer small complete builds over many unfinished experiments.
7. Classify repos before deleting anything.
8. Never expose private operational secrets.

## Daily Build Process

```text
3. Backup
4. Build
5. Test
6. Save result

```
